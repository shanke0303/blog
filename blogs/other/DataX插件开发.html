    DataX插件开发

    shanke 2021/3/9 datax

    # 1.下载DataX

    DataX Github：https://github.com/alibaba/DataX

    git clone https://github.com/alibaba/DataX.git
    1

    # 2.从Idea打开

    # 3.修改pom.xml[可选]

    根目录目录下的文件pom.xml ，把<modules>标签下原本不需要的reader 和 writer 注释掉。

    # 4.开发一个reader（writer同理）

    新建一个模块，DataX目录右键：New > Module...。

    左侧指定Maven。

    输入名称。注意：位置在/DataX下面。

    从其它reader模块复制一个pom.xml覆盖到自己的模块下。

    • 修改<artifactId>和<name>。
    • <dependencies>中删除不需要的依赖，添加自己要用的依赖。

    从其它reader模块main目录下复制一份assembly(包含子文件package.xml)到自己的模块main目录下。

    • 相关的路径名称修改为自己的。

    编写自己的reader，也可以从其它模块复制修改。

    DataX插件开发官方文档：https://github.com/alibaba/DataX/blob/master/dataxPluginDev.md

    从其它reader模块resource目录下复制plugin.json和plugin_job_template.json到自己模块中，并按自己的情况修改。

    • plugin.json，告诉DataX如何找到你的插件。

    • plugin_job_template.json，生成作业配置模板时使用。运行datax.py -r <reader name> -w <writer name>会根据这个文件生成作业配置模板。

    # 5.打包

    根目录目录下的文件package.xml ，把<fileSets>标签下原本不需要的reader 和 writer 注释掉。把自己的木块添加进去。

    点开右侧边栏Maven，禁用test，点击package。

    在模块target/data下可以找的打包好的插件。

    # 6.常见问题

    解决：

    • mac系统会自动生成一个.DS_Store的系统文件，不可见。而datax默认plugin/reader中都是文件夹，没有做操作系统这种层面的差异处理。所以需要手动把错误路径下的.DS_Store文件删除即可。
    cd datax
find ./ -name '.DS_Store' -print0 | xargs -0 rm -fr
